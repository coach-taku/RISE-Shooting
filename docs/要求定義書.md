# RISE Shooting　記録・分析アプリ 要求定義書

## 0. アプリの概要
- **目的:** バスケットボール選手一人一人のシューティング記録データを蓄積し、それぞれの得意なエリアや成長を客観的に把握するため。
- **主な機能:**
  - 選手ごとのシューティング記録（エリア別の試投数・成功数）の入力・蓄積機能
  - エリアごとの成功確率、打った本数、決めた本数のフィードバック（視覚的なゾーン別マップ・ヒートマップ表示）
  - チーム内で選手が競い合える仕組み（ランキングやチーム内比較機能）
  - 選択した期間（1週間、1ヶ月、指定期間など）ごとのデータ集計・絞り込み機能
  - 指定されたコート図に基づく詳細なエリア分け（2P・3P各種ゾーン）

## 1. 🏟️ Context（前提・文脈・使用シーン）
- **誰が使うか:**
  - 選手: スマートフォン操作に慣れている高校生バスケットボール部員
  - コーチ: 選手のスタッツを正確に把握し、個別の指導や戦術に活かしたい指導者
- **いつ・どこで使うか:**
  - 練習（シューティングメニュー）の合間や練習直後、体育館にて各自のスマートフォンで入力
  - 自主練の振り返りとして、帰宅中や自宅にて入力・確認
- **現在の課題（想定される状態）:**
  - シューティングの結果が選手の感覚や記憶に頼りがちで、得意・不得意なエリアがデータとして可視化されていない。

## 2. 🤕 Pain（苦痛・解決したい課題）
- 「どのエリアからのシュートが得意か/苦手か」が客観的な数値で示されていないため、効率的なスキルアップや試合での適切なシュートセレクションの指導が難しい。
- 日々の反復練習になりがちなシューティングにおいて、目に見える成長実感やチームメイトとの競争要素がないと、選手のモチベーションが維持しにくい。
- アプリを一度閉じると次回起動時に毎回ログインが必要で、日々の入力の手間が発生している。→ **v4.2 で解消済み**（Next.js 16 規約の `proxy.ts` による Cookie セッション永続化を実装。`middleware.ts` は Next.js 16 で非推奨のため `proxy.ts` へ移行）
- カレンダーから特定の日付を選択して記録を入力することができず、過去の記録の遡り入力や特定日の追記がスムーズに行えなかった。→ **v6 で解消済み**（インラインカレンダーUIを追加。過去・未来の制限なく任意の日付を選択して記録・追記が可能になった）
- バグにより過去の入力データに誤りが発生した場合に、選手が自身で修正・削除できる手段がなかった。→ **v8 で解消済み**（シュート記録入力画面に過去データの修正・削除機能を追加。カレンダーで過去の日付を選択すると既存データが表示され、修正または削除できるようになった）

## 3. 🎨 Philosophy（思想・美学・こだわり）
- **客観視による自己成長:** 自分のスタッツ（ホットゾーン・コールドゾーン）を可視化することで、自ら課題を発見し、試合中の的確な判断（シュートセレクション）ができる「自己管理能力」を養う。
- **切磋琢磨する環境作り:** チーム内でのランキングや比較機能を通じて、日々のシューティング練習にゲーム性と緊張感をもたせ、チーム全体のシュート力を底上げする。

## 4. 📐 Specifications（機能・制約）
### A. 入力項目（Input）
入力の負担を減らすため、項目は最小限かつ直感的にします。
- **対象データ:**
  - 実施日（期間集計用）— **インラインカレンダーで任意の日付を選択可能（v6改善）**
  - **各エリアでのシュート結果（10本単位での入力）**
    - 「10本中何本成功したか」を入力する形式。
    - 基準を10本に統一することで、入力の手間を省きつつ、練習ごとの確率のブレを標準化して蓄積しやすくする。
- **エリア分け（添付スクリーンショットに基づく区分）:**
  - ゴール下（ペイント内）
  - ショート〜ミドルレンジ（左・中央・右などの複数ゾーン）
  - 3Pエリア（左コーナー、左ウイング、トップ、右ウイング、右コーナーなど）

### B. 出力・閲覧（Output）
- **個人成績画面（ショットチャート）:**
  - 添付画像のようなコートマップを用いた「ゾーン別」「ヒートマップ」表示
  - 各エリアの「成功数 / 試投数」および「成功率（%）」の可視化
  - 2P、3P、FG全体、FTなどのカテゴリ別成績サマリー表示
- **チーム内比較・ランキング画面:**
  - シュート成功率、成功本数などのチーム内ランキング
- **期間集計機能:**
  - 今週、今月、全期間、または任意の期間を選択し、その期間内のみの合算データをマップ上に瞬時に反映

### C. やらないこと（Scope Out）
- 試合中のリアルタイムなトラッキング（本アプリは日々のシューティング練習の記録・蓄積を主目的とするため、複雑な試合用スタッツアプリではなく、シンプルに練習の記録ができることを優先する）。
- 重い動画のアップロードやフォーム解析機能。

## 5. 🚀 Workflow（運用イメージ）
1. **練習実施:** チーム練習または自主練で、各エリア10本単位のシューティングメニューを実施。
2. **データ入力:** 練習後、選手が各自のスマホからその日のエリア別の「10本中の成功数」を入力。ログイン状態が維持されているため、アプリを開くとすぐに入力画面へ進める。
3. **即時フィードバック:** 入力後すぐに自分のスタッツ画面が更新され、本日の確率や期間累計のマップを確認。
4. **チーム内競争:** リーダーボード（ランキング）を見て、チームメイトの記録を確認し、翌日の練習へのモチベーションを高める。
5. **コーチの確認:** コーチはダッシュボードから全選手のデータやヒートマップを確認し、「〇〇は右ウイングの確率が上がってきたね」など、客観的データに基づいた個別アドバイスを行う。
